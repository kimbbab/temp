사례: 대화로 만드는 게임

  • 유튜브 화제의 영상: "Vibe Coding a Game"
  • 복잡한 알고리즘 설계 없이
  • "장애물을 피해 점수 얻는 게임 만들어줘"
  • 채팅 몇 번으로 완성되는 결과물
3 / 15

실제 활용 01
PPT 자동화

과거: 인터넷에서 스크립트를 검색하고 내 상황에 맞춰 수정하느라 몇 시간을 허비했습니다.

현재: "폴더 내 모든 PPT를 PDF로 변환하고 암호를 걸어줘"라고 말하면 AI가 10초 만에 코드를 완성합니다.

4 / 15

실제 활용 02
QR 코드 일괄 생성

  • 구글 시트 웹 주소 → QR 변환
  • 이미지 자동 다운로드 스크립트
  • 단순 반복 작업을 0분으로 단축
  • 비개발자도 관리 도구를 직접 제작
5 / 15

실제 활용 03
동영상 썸네일 추출

  • ffmpeg를 활용한 배치파일(.bat) 제작
  • 여러 동영상의 1.5초 구간 썸네일 자동 추출
  • 비디오 편집 프로그램 없이 대량 작업 완수
  • 단순 수작업 노동을 AI 코드로 완벽히 자동화
6 / 15

실제 활용 04
대량 파일명 일괄 변경

  • 폴더 내 대량의 파일명 목록 추출
  • 구글 시트(엑셀)에 데이터 자동 정리
  • 사내 규칙에 맞게 파일명 변형 규칙 적용
  • 버튼 한 번으로 전체 파일명 일괄 변경 스크립트 작성
7 / 15

실제 활용 05
AIDT 게임 제작

  • 편집자 (비개발자): 창의적인 게임 아이디어 기획
  • AI (LLM): 기획을 바탕으로 HTML 게임 초안 작성
  • 개발자: 초안을 넘겨받아 실제 게임으로 개발 고도화
  • 기획 의도가 코드로 즉시 구현되는 새로운 경험
8 / 15

실제 활용 06
이 발표 자료 제작

  • 사실 이 발표 자료도 Gemini를 이용하여 만들었습니다.
  • "팀장급 대상 발표 자료를 만들어줘"라는 자연어 대화가 그 시작이었습니다.
  • 단순 텍스트 초안부터 지금 보시는 HTML 슬라이드 화면 구현까지
  • 이 자료를 만드는 과정 자체가 완벽한 '바이브 코딩'의 사례입니다.

시사점 01: 직접 해보십시오

  • 무작정 "AI 도입해!"라고 외치는 것은 피해야 할 방향입니다
  • '될 것 같은데 해봐' 대신 '직접 해보니 되니까 시켜라'
  • 직접 해봐야 AI의 가능성과 한계를 정확히 이해할 수 있습니다
  • 팀장님들부터 AI 리터러시를 갖춰야 올바른 업무 지시가 가능합니다

현실과 한계점

  • 비전문가용 마법? 대단해 보이지만 복잡한 구조 설계에는 분명한 한계가 존재합니다.
  • 잠잠해진 열풍: "다 되는 줄 알았는데" 생각보다 마음대로 되지 않아 포기하는 사례도 많습니다.
  • 전문가의 영역: 개발자가 직접 Cursor AI 등을 활용할 때 10배 더 강력한 도구가 됩니다.
  • 지속 가능한 발전: 결국 비개발자도 코딩의 흐름을 이해하려는 기초 지식이 동반되어야 합니다.
